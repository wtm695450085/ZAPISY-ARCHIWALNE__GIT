Co to jest liniowy klasyfikator? Dwa najczciej nadzorowane zadania edukacyjne to regresja liniowa i liniowy klasyfikator. Regresja liniowa przewiduje warto, a klasyfikator liniowy przewiduje klas. Ten samouczek koncentruje si na klasyfikatorze liniowym.

Problemy z klasyfikacją stanowią około 80 procent zadania uczenia maszynowego. Klasyfikacja ma na celu przewidzenie prawdopodobieństwa każdej klasy na podstawie zestawu danych wejściowych.¶

Etykieta (tj. Zmienna zalena) jest wartoci dyskretn, zwan klas.

  • Jeli etykieta ma tylko dwie klasy, algorytm uczenia si jest klasyfikatorem binarnym.
  • Wieloklasowy klasyfikator obsuguje etykiety z wicej ni dwiema klasami.

Na przykad typowym problemem z klasyfikacj binarn jest przewidywanie prawdopodobiestwa, e klient dokona drugiego zakupu. Przewidywanie rodzaju zwierzcia wywietlanego na zdjciu jest problemem klasyfikacji wieloklasowej, poniewa istniej wicej ni dwie odmiany zwierzt.

Cz teoretyczna tego samouczka kadzie gwny nacisk na klas binarn. Dowiesz si wicej o funkcji wyjciowej wieloklasowej w przyszym samouczku.

W tym samouczku nauczysz si - Co to jest liniowy klasyfikator?

def print_transformation(feature = "age", continuous = True, size = 2): 
    #X = fc.numeric_column(feature)
    ## Create feature name
    feature_names = [
    feature]

    ## Create dict with the data
    d = dict(zip(feature_names, [df_train[feature]]))

    ## Convert age
    if continuous == True:
        c = tf.feature_column.numeric_column(feature)
        feature_columns = [c]
    else: 
        c = tf.feature_column.categorical_column_with_hash_bucket(feature, hash_bucket_size=size) 
        c_indicator = tf.feature_column.indicator_column(c)
        feature_columns = [c_indicator]
    
## Use input_layer to print the value
    input_layer = tf.feature_column.input_layer(
        features=d,
        feature_columns=feature_columns
        )
    ## Create lookup table
    zero = tf.constant(0, dtype=tf.float32)
    where = tf.not_equal(input_layer, zero)
    ## Return lookup tble
    indices = tf.where(where)
    values = tf.gather_nd(input_layer, indices)
    ## Initiate graph
    sess = tf.Session()
    ## Print value
    print(sess.run(input_layer))

def get_input_fn(data_set, num_epochs=None, n_batch = 128, shuffle=True):
    return tf.estimator.inputs.pandas_input_fn(
       x=pd.DataFrame({k: data_set[k].values for k in FEATURES}),
       y = pd.Series(data_set[LABEL].values),
       batch_size=n_batch,   
       num_epochs=num_epochs,
       shuffle=shuffle)

Krok 6: To evaluate the performance of model¶

ulepszanie modelu

Teraz, gdy masz model testowy, moesz sprbowa go ulepszy, czyli zwikszy dokadno. W poprzednim samouczku nauczye si, jak poprawi moc przewidywania za pomoc terminu interakcji. W tym samouczku ponownie przyjrzysz si temu pomysowi, dodajc do regresji termin wielomianowy.

Regresja wielomianowa jest instrumentalna, gdy w danych występuje nieliniowość. Istnieją dwa sposoby uchwycenia nieliniowości danych.¶

  • Dodaj zmienn wielomianow
  • Podziel zmienn cig na zmienn jakociow

Sposób 1: Dodaj zmienną wielomianową¶

Tworzenie zmiennej syntetycznej w celu poprawy modelu

Na poniszym obrazku moesz zobaczy, czym jest regresja wielomianowa. Jest to rwnanie ze zmiennymi X o rnej mocy. Regresja wielomianowa drugiego stopnia ma dwie zmienne, X i X podniesione do kwadratu. Trzeci stopie ma trzy zmienne, X, X^2 i X^3

Poniej skonstruowalimy wykres z dwiema zmiennymi, X i Y. Oczywiste jest, e zwizek nie jest liniowy. Jeli dodamy regresj liniow, moemy zobaczy, e model nie jest w stanie uchwyci wzoru (lewy obraz).

Teraz spjrz na lewy obrazek z poniszego obrazka, dodalimy piciokresow regresj

(to znaczy y = x + x^2 + x^3 + x^4 + x^5. Model teraz przechwytuje znacznie lepszy wzr. To jest moc regresji wielomianowej.

Tworzenie zmiennej syntetycznej w postaci kwadratu zmiennej 'age'¶

Wiek nie ma zwizku liniowego z dochodami. Wczesny wiek moe mie stay dochd bliski zeru, poniewa dzieci lub modzi ludzie nie pracuj. Nastpnie zwiksza si w wieku produkcyjnym i maleje podczas przechodzenia na emerytur. Zazwyczaj jest to ksztat odwrcony-U. Jednym ze sposobw na uchwycenie tego wzorca jest dodanie potgi drugiej do regresji.

Zobaczmy, czy zwikszy to dokadno.

Musisz doda t now funkcj do zestawu danych i na licie funkcji cigej.

Dodajesz now zmienn do zestawu danych treningowych i testowych, wic wygodniej jest napisa funkcj.

def square_var(df_t, df_te, var_name = 'age'):
    df_t['new'] = df_t[var_name].pow(2) 
    df_te['new'] = df_te[var_name].pow(2) 
    return df_t, df_te

Funkcja definiuje pojawienie się kolumny 'new' w biorze testowymi i zbiorze treningowym¶

Funkcja ma 3 argumenty:

df_t: zdefiniuj zestaw treningowy df_te: zdefiniuj zestaw testowy var_name = 'age': Zdefiniuj zmienn do transformacji Moesz uy obiektu pow (2) do kwadratu zmiennej wiek. Zauwa, e nowa zmienna nosi nazw nowa

ULEPSZANIE MODELU

Sposób 2: Podzielenie zmiennej ciągłą na zmienną jakościową¶

>>> Bucketization¶

Funkcja: wiadrowania lub inaczej tworzenie klas w obrbie zmiennej cigej

Jak widzielicie wczeniej, liniowy klasyfikator nie jest w stanie poprawnie uchwyci wzorca wieku i dochodu. To dlatego, e uczy si pojedynczej wagi dla kadej funkcji. Aby uatwi klasyfikatorowi, jedn z rzeczy, ktre moesz zrobi, jest umieszczenie tej funkcji w wiadrze. Wiadro przeksztaca funkcj numeryczn w kilka okrelonych na podstawie zakresu, w ktrym si mieci, a kada z tych nowych funkcji wskazuje, czy wiek osoby mieci si w tym zakresie.

Dziki tym nowym funkcjom model liniowy moe uchwyci zwizek, uczc si rnych wag dla kadego segmentu.

Dzielimy zmienną ciągłą 'age' na grupy wiekowe: [18, 25, 30, 35, 40, 45, 50, 55, 60, 65]¶

In [33]:
age = tf.feature_column.numeric_column('age')
age_buckets = tf.feature_column.bucketized_column(
    age, boundaries=[18, 25, 30, 35, 40, 45, 50, 55, 60, 65])

>>> Tworzenie zmiennej syntetycznej z interakcji pomiędzy zmiennymi niezależnymi¶

Funkcja: interakcji

Wiesz ju, e wiek nie jest liniowy z dochodami. Innym sposobem na ulepszenie modelu jest interakcja (interaction). Wedug TensorFlow jest to skrzyowanie funkcji. Przekraczanie cech jest sposobem na tworzenie nowych cech, ktre s kombinacjami istniejcych, ktre mog by pomocne dla klasyfikatora liniowego, ktry nie moe modelowa interakcji midzy cechami.

Wiek moesz podzieli wedug innej funkcji, takiej jak edukacja. Oznacza to, e niektre grupy mog mie wysokie dochody, a inne niskie (Pomyl o doktoracie).

Sposb na tworzenie nowych zmiennych, ktre s kombinacjami istniejcych, ktre mog by pomocne dla klasyfikatora liniowego, ktry pierwotnie nie moe modelowa interakcji midzy cechami.

  • hash_bucket_size = 1000 ustawiamy na 1000, poniewa nie znasz dokadnej liczby grup

Hyperparameter:Lasso & Ridge¶

Add regularization parameter¶

Twj model moe cierpie z przeuczenia lub niedouczenia (overfitting or underfitting) .

overfitting: model nie jest w stanie uogólnić prognozy na nowe dane¶

underfitting: Niedopasowanie : model nie może przechwycić wzorca danych. np. regresja liniowa, gdy dane są nieliniowe¶

Aby zapobiec nadmiernemu dopasowaniu (overfitting), regularyzacja daje moliwo kontrolowania takiej zoonoci i czyni j bardziej uoglnion. Istniej dwie techniki regularyzacji:

L1: Lasso
L2: Ridge

W TensorFlow moesz doda te dwa hiperparametry w optymalizatorze.

Na przykad, im wyszy jest hiperparametr L2: Ridge, waga jest zwykle bardzo niska i bliska zeru. Dopasowana linia bdzie bardzo paska, podczas gdy

L2 bliskie zeru oznacza, e wagi s bliskie regularnej regresji liniowej.

Moesz samodzielnie rcznie, wyprbowa inn warto hiperparametrw i sprawdzi, czy moesz zwikszy poziom dokadnoci.

Pamitaj, e jeli zmienisz hiperparametr, musisz usun folder obecny / train4 zmieni jego nazw, w przeciwnym razie model rozpocznie si od wczeniej wyuczonego modelu.

Twprzymyu z udziam L1 i L2 nowy model - model_regu gdzie:

   -   L1 = 0.9 
   -   L2 = 5
