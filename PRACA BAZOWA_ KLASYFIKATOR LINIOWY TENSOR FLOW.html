Co to jest liniowy klasyfikator? Dwa najczęściej nadzorowane zadania edukacyjne to regresja liniowa i liniowy klasyfikator. Regresja liniowa przewiduje wartość, a klasyfikator liniowy przewiduje klasę. Ten samouczek koncentruje się na klasyfikatorze liniowym.

Problemy z klasyfikacją stanowią około 80 procent zadania uczenia maszynowego. Klasyfikacja ma na celu przewidzenie prawdopodobieństwa każdej klasy na podstawie zestawu danych wejściowych.¶

Etykieta (tj. Zmienna zależna) jest wartością dyskretną, zwaną klasą.

  • Jeśli etykieta ma tylko dwie klasy, algorytm uczenia się jest klasyfikatorem binarnym.
  • Wieloklasowy klasyfikator obsługuje etykiety z więcej niż dwiema klasami.

Na przykład typowym problemem z klasyfikacją binarną jest przewidywanie prawdopodobieństwa, że klient dokona drugiego zakupu. Przewidywanie rodzaju zwierzęcia wyświetlanego na zdjęciu jest problemem klasyfikacji wieloklasowej, ponieważ istnieją więcej niż dwie odmiany zwierząt.

Część teoretyczna tego samouczka kładzie główny nacisk na klasę binarną. Dowiesz się więcej o funkcji wyjściowej wieloklasowej w przyszłym samouczku.

def print_transformation(feature = "age", continuous = True, size = 2): 
    #X = fc.numeric_column(feature)
    ## Create feature name
    feature_names = [
    feature]

    ## Create dict with the data
    d = dict(zip(feature_names, [df_train[feature]]))

    ## Convert age
    if continuous == True:
        c = tf.feature_column.numeric_column(feature)
        feature_columns = [c]
    else: 
        c = tf.feature_column.categorical_column_with_hash_bucket(feature, hash_bucket_size=size) 
        c_indicator = tf.feature_column.indicator_column(c)
        feature_columns = [c_indicator]
    
## Use input_layer to print the value
    input_layer = tf.feature_column.input_layer(
        features=d,
        feature_columns=feature_columns
        )
    ## Create lookup table
    zero = tf.constant(0, dtype=tf.float32)
    where = tf.not_equal(input_layer, zero)
    ## Return lookup tble
    indices = tf.where(where)
    values = tf.gather_nd(input_layer, indices)
    ## Initiate graph
    sess = tf.Session()
    ## Print value
    print(sess.run(input_layer))

def get_input_fn(data_set, num_epochs=None, n_batch = 128, shuffle=True):
    return tf.estimator.inputs.pandas_input_fn(
       x=pd.DataFrame({k: data_set[k].values for k in FEATURES}),
       y = pd.Series(data_set[LABEL].values),
       batch_size=n_batch,   
       num_epochs=num_epochs,
       shuffle=shuffle)

Krok 6: To evaluate the performance of model¶

ulepszanie modelu

Teraz, gdy masz model testowy, możesz spróbować go ulepszyć, czyli zwiększyć dokładność. W poprzednim samouczku nauczyłeś się, jak poprawić moc przewidywania za pomocą terminu interakcji. W tym samouczku ponownie przyjrzysz się temu pomysłowi, dodając do regresji termin wielomianowy.

Regresja wielomianowa jest instrumentalna, gdy w danych występuje nieliniowość. Istnieją dwa sposoby uchwycenia nieliniowości danych.¶

  • Dodaj zmienną wielomianową
  • Podziel zmienną ciągłą na zmienną jakościową

Sposób 1: Dodaj zmienną wielomianową¶

Tworzenie zmiennej syntetycznej w celu poprawy modelu

Na poniższym obrazku możesz zobaczyć, czym jest regresja wielomianowa. Jest to równanie ze zmiennymi X o różnej mocy. Regresja wielomianowa drugiego stopnia ma dwie zmienne, X i X podniesione do kwadratu. Trzeci stopień ma trzy zmienne, X, X^2 i X^3

Poniżej skonstruowaliśmy wykres z dwiema zmiennymi, X i Y. Oczywiste jest, że związek nie jest liniowy. Jeśli dodamy regresję liniową, możemy zobaczyć, że model nie jest w stanie uchwycić wzoru (lewy obraz).

Teraz spójrz na lewy obrazek z poniższego obrazka, dodaliśmy pięciokresową regresję

(to znaczy y = x + x^2 + x^3 + x^4 + x^5. Model teraz przechwytuje znacznie lepszy wzór. To jest moc regresji wielomianowej.

Tworzenie zmiennej syntetycznej w postaci kwadratu zmiennej 'age'¶

Wiek nie ma związku liniowego z dochodami. Wczesny wiek może mieć stały dochód bliski zeru, ponieważ dzieci lub młodzi ludzie nie pracują. Następnie zwiększa się w wieku produkcyjnym i maleje podczas przechodzenia na emeryturę. Zazwyczaj jest to kształt odwrócony-U. Jednym ze sposobów na uchwycenie tego wzorca jest dodanie potęgi drugiej do regresji.

Zobaczmy, czy zwiększy to dokładność.

Musisz dodać tę nową funkcję do zestawu danych i na liście funkcji ciągłej.

Dodajesz nową zmienną do zestawu danych treningowych i testowych, więc wygodniej jest napisać funkcję.

In [26]:
def square_var(df_t, df_te, var_name = 'age'):
    df_t['new'] = df_t[var_name].pow(2) 
    df_te['new'] = df_te[var_name].pow(2) 
    return df_t, df_te

Funkcja definiuje pojawienie się kolumny 'new' w biorze testowymi i zbiorze treningowym¶

Funkcja ma 3 argumenty:

df_t: zdefiniuj zestaw treningowy df_te: zdefiniuj zestaw testowy var_name = 'age': Zdefiniuj zmienną do transformacji Możesz użyć obiektu pow (2) do kwadratu zmiennej wiek. Zauważ, że nowa zmienna nosi nazwę „nowa”

Teraz, gdy została napisana funkcja square_var, możesz tworzyć nowe zestawy danych.

ULEPSZANIE MODELU

Sposób 2: Podzielenie zmiennej ciągłą na zmienną jakościową¶

>>> Bucketization¶

Funkcja: wiadrowania lub inaczej tworzenie klas w obrębie zmiennej ciągłej

Jak widzieliście wcześniej, liniowy klasyfikator nie jest w stanie poprawnie uchwycić wzorca wieku i dochodu. To dlatego, że uczy się pojedynczej wagi dla każdej funkcji. Aby ułatwić klasyfikatorowi, jedną z rzeczy, które możesz zrobić, jest umieszczenie tej funkcji w wiadrze. Wiadro przekształca funkcję numeryczną w kilka określonych na podstawie zakresu, w którym się mieści, a każda z tych nowych funkcji wskazuje, czy wiek osoby mieści się w tym zakresie.

Dzięki tym nowym funkcjom model liniowy może uchwycić związek, ucząc się różnych wag dla każdego segmentu.

Dzielimy zmienną ciągłą 'age' na grupy wiekowe: [18, 25, 30, 35, 40, 45, 50, 55, 60, 65]¶

In [33]:
age = tf.feature_column.numeric_column('age')
age_buckets = tf.feature_column.bucketized_column(
    age, boundaries=[18, 25, 30, 35, 40, 45, 50, 55, 60, 65])

>>> Tworzenie zmiennej syntetycznej z interakcji pomiędzy zmiennymi niezależnymi¶

Funkcja: interakcji

Wiesz już, że wiek nie jest liniowy z dochodami. Innym sposobem na ulepszenie modelu jest interakcja (interaction). Według TensorFlow jest to skrzyżowanie funkcji. Przekraczanie cech jest sposobem na tworzenie nowych cech, które są kombinacjami istniejących, które mogą być pomocne dla klasyfikatora liniowego, który nie może modelować interakcji między cechami.

Wiek możesz podzielić według innej funkcji, takiej jak edukacja. Oznacza to, że niektóre grupy mogą mieć wysokie dochody, a inne niskie (Pomyśl o doktoracie).

Sposób na tworzenie nowych zmiennych, które są kombinacjami istniejących, które mogą być pomocne dla klasyfikatora liniowego, który pierwotnie nie może modelować interakcji między cechami.

  • hash_bucket_size = 1000 ustawiamy na 1000, ponieważ nie znasz dokładnej liczby grup

Hyperparameter:Lasso & Ridge¶

Add regularization parameter¶

Twój model może cierpieć z przeuczenia lub niedouczenia (overfitting or underfitting) .

overfitting: model nie jest w stanie uogólnić prognozy na nowe dane¶

underfitting: Niedopasowanie : model nie może przechwycić wzorca danych. np. regresja liniowa, gdy dane są nieliniowe¶

Aby zapobiec nadmiernemu dopasowaniu (overfitting), regularyzacja daje możliwość kontrolowania takiej złożoności i czyni ją bardziej uogólnioną. Istnieją dwie techniki regularyzacji:

L1: Lasso
L2: Ridge

W TensorFlow możesz dodać te dwa hiperparametry w optymalizatorze.

Na przykład, im wyższy jest hiperparametr L2: Ridge, waga jest zwykle bardzo niska i bliska zeru. Dopasowana linia będzie bardzo płaska, podczas gdy

L2 bliskie zeru oznacza, że wagi są bliskie regularnej regresji liniowej.

Możesz samodzielnie ręcznie, wypróbować inną wartość hiperparametrów i sprawdzić, czy możesz zwiększyć poziom dokładności.

Pamiętaj, że jeśli zmienisz hiperparametr, musisz usunąć folder obecny / train4 zmienić jego nazwę, w przeciwnym razie model rozpocznie się od wcześniej wyuczonego modelu.

   -   L1 = 0.9 
   -   L2 = 5
